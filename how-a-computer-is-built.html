How a computer is built from switches

At the very bottom, a computer is just switches turning each other on and off. Everything a computer does is built from them by plain, understandable steps, no magic in the wiring, though the switch itself rests on some very deep physics.

1A wire is on or off

At the bottom, a computer is wires carrying electricity. Voltage is the electrical push that makes that electricity flow, like water pressure in a pipe. A wire is in one of two states: voltage is present, or it is not. We call these two states 1 and 0.

1
voltage present (on)
0
no voltage (off)

2A switch you can control

Now we need something that can act on those 1s and 0s. That something is a switch.

You already know a switch: a light switch opens or closes a connection. A computer's switch, called a transistor, does the same thing, with one important difference. It is controlled by electricity, not by a finger.

A transistor has a control wire. When the control wire is 1, the switch closes and current flows through. When the control wire is 0, the switch opens and current stops.

That is the one fact everything else is built on: a switch that another wire can turn on and off.

How a transistor physically does this, how a sliver of silicon can act as a switch at all, is a question of semiconductor physics, which itself rests on quantum mechanics. That is a layer below this one, and a deep one. We will not explain it here; we treat the switch as given.

3Wire switches into logic

Now we wire switches together and watch when current can get through. Two arrangements matter. (Both use the controlled switch from section 2: a = 1 means that switch's control is on, so the switch is closed.)

Two switches in series

Current reaches the output only when both switches are closed.

Two switches in parallel

Current reaches the output when either switch is closed, here, a alone is enough.

Read a closed switch as an input of 1. Then those two arrangements are two simple rules: current gets through only when both inputs are 1, or when either input is 1. Rules like these are useful enough that we name them. "Both" is called AND. "Either" is called OR. A third basic rule, NOT, simply flips its input: 1 becomes 0, and 0 becomes 1.

These named rules (AND, OR, NOT) are the logic gates. They are the first level built on top of the bare switch, and the point where the physical world meets pure logic: a gate is electricity, while AND, OR, and NOT are ideas from logic. From here up, we can stop thinking about wires and current, and think in terms of these rules.

4From logic, build arithmetic

The same 1s and 0s can also stand for numbers, 0 and 1 are the two digits of binary. Adding two of them is just another input-to-output rule, exactly like AND and OR, so the same kind of gate can do it. Start with the smallest case: adding two single bits.

absumcarry
0000
0110
1010
1101

Two bits in; a sum bit and a carry bit out.

Three of these are easy: 0+0, 0+1, 1+0. The interesting one is the last row. In a world of 1s and 0s, 1+1 is 2, and 2 is written as 10. So the result has a 0 in this column and a 1 carried into the next column, exactly like carrying when you add 7 + 5 by hand.

Two gates produce exactly that table:

  • The carry is 1 only when both inputs are 1, which is just an AND.
  • The sum is 1 when the two inputs are different (a 0 and a 1). That "1 when they differ" rule is common enough to have its own name: XOR (exclusive or). XOR is itself built from AND, OR, and NOT, so we are still using only the basic gates.

An XOR gate gives the sum; an AND gate gives the carry. Both read the same two inputs, a and b.

A circuit like this is called a half-adder. To add longer numbers, you use one circuit per column and pass each column's carry into the next. (Every column after the first also has to take in a carry, so it uses a slightly bigger version called a full adder: same idea, one extra input.) So "a computer does math" is not magic. It is gates, which are switches.

5From logic, build memory

Gates can also remember. Here is how, built up in a few steps.

First, one more gate. A NOR gate is an OR followed by a NOT: its output is 1 only when both inputs are 0, and 0 if either input is 1.

Now take two NOR gates and feed each one's output back into an input of the other. Give each gate one spare input from the outside, called S (set) and R (reset). This loop is called an S-R latch.

Two NOR gates, each output feeding back into the other (the green wires).

Watch it hold a bit:

  • Put a 1 on S (with R at 0): the output Q becomes 1.
  • Put a 1 on R (with S at 0): Q becomes 0.
  • Set both S and R back to 0: the latch keeps the value it last had. Each gate feeds the other, so the two hold each other steady. That held value is the stored bit.

So memory is not hand-waving. It is two gates wired in a loop: S writes a 1, R writes a 0, and with both off the loop simply holds. (The second output, NOT Q, is always the opposite of Q; we can ignore it.)

That is one bit. A loop like this is how the fast memory nearest the processor, its registers and caches, holds bits, though real cells use a more refined version of the circuit, not this exact one. A computer's main memory works differently again: it stores each bit as a tiny charge that leaks away and has to be topped up. Those are topics for later. The idea to keep from here is the general one: a circuit can hold a value by feeding its output back into its input.

6The clock breaks time into steps

A circuit is not instant. When its inputs change, the new voltages spread through the wires, close to the speed of light, but not quite, and the circuit settles into a new state. It is an analog process: loosely, like water flooding through a network of pipes until it comes to rest. Signals along different paths settle at slightly different moments, so for a brief stretch after any change, the values are still racing around and have not yet resolved.

That unsettled stretch is a problem for a circuit that feeds back on itself, like the latch. If the parts react to each other while the values are still moving, the whole thing can land in the wrong state, or never settle at all. So we want the parts to change only at set moments, after the values have settled, not the instant a stray signal arrives.

So we add one shared signal that pulses steadily: on, off, on, off. Every part is built to change only on the pulse. Between pulses nothing changes, and the signals have time to settle. On the next pulse, every part takes one step at once. That shared pulse is the clock.

Now two things come in clean, separate pieces instead of a continuous flow: values, where a wire is either 1 or 0, and time, which is either a tick or the gap between ticks. Breaking time into steps is what lets a circuit do one thing, then the next, in order, instead of everything happening at once.

The pulse itself comes from a quartz crystal, which vibrates at a fixed rate when current runs through it. The circuit counts those vibrations to keep time. So even the steady tick has an analog effect underneath it.

7The parts of a CPU

We now have the basic parts a CPU is built from: gates and adders that do logic and arithmetic, memory cells that hold bits, and a clock to step everything in order. Bundle the small parts into bigger ones:

  • Many adding circuits together make an arithmetic unit, a block that can add, compare, and do other operations on numbers.
  • Many memory cells together make registers, small, fast places to hold the numbers being worked on.
  • A control part decides, on each clock tick, which operation to do and where the numbers come from.

But having the parts is not having a CPU. A CPU is these parts wired together in a particular way, connected so they can pass numbers to each other, and controlled so that each tick carries out one step of an instruction. 8The power of layers of abstraction

Step back and look at the climb. The important thing about it is not any single level, but that each level let us stop thinking about the one below.

Once we had a switch, we stopped thinking about electrons. Once we had a logic gate, we stopped thinking about switches. Once we had an adder, we stopped thinking about gates. Each level hides the one below and gives us a simpler thing to work with. This hiding is called abstraction, and it is the only reason a person can build something out of billions of parts.

1 switch → billions of switches on one chip

A modern chip really does have billions of switches. No person could hold all of that in mind at once, and we never have to. Each part is simple, and each layer lets us ignore everything beneath it. The complexity is in the number of parts and the way they are arranged, not in any single piece.

The clean picture does leak. At very high speeds and very small sizes, the messy analog reality (noise, heat, timing) comes back, and handling it is a large part of why designing chips is hard. And there is still a layer beneath this one: the physics of how a single switch works (semiconductor physics, resting on quantum mechanics), which we treated as given. So if you are looking for something magical here, it is not in the wiring we built. It is down in that bottom layer, the physics we stood on without explaining. Every layer rests on another.

That is the same lesson as the rest of this series, seen from the floor: we manage overwhelming complexity by building it in layers, where each layer hides the one below.

Endnote: 0 and 1 are a choice

Back in section 1, we said a wire is either 1 or 0. That was a simplification.

Real electricity is not that tidy. The voltage on a wire is a messy, continuous thing. We choose two ranges, call them "low" and "high", and agree to treat every wire as either 0 or 1. Snapping a messy signal to two clean states is what makes computers reliable: small amounts of noise stop mattering. And that choice is the first abstraction of all: it hides the messy analog reality so every level above can work in clean 0s and 1s.
